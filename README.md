Веб-сервис для подачи показаний счетчиков отопления, горячей и холодной воды. Реализовано хранение данных с помощью коллекций.
# Описание
Показания можно подавать один раз в месяц.
Ранее поданые показания редактировать запрещено.
Последние поданые показания считаются актуальными.
Пользователь может видеть только свои показания, администратор может видеть показания всех пользователей.
Создайте реализацию, которая соответствует описанным ниже требованиям и ограничениям.

# Сервис управления данными о потреблении воды

## Описание
Класс Service представляет собой утилиту для управления учетными записями пользователей и данными о потреблении воды. Включает в себя методы для регистрации пользователя, аутентификации, добавления информации о потреблении воды, извлечения данных о потреблении и проверки существования данных для определенного пользователя и месяца.

## Использование
1. **Инициализация сервиса**
   ```java
   Service service = new Service();
<h3>Регистрация нового пользователя</h3>

boolean isRegistered = service.registerUser("username", "password");
<br>
<h3>Аутентификация пользователя</h3>

User authenticatedUser = service.authenticateUser("username", "password");
<br>
<h3>Добавление информации о потреблении воды</h3>

service.addUserAndWaterInfo(user, "month", hotWaterConsumption, coldWaterConsumption);
<br>
<h3>Получение данных о потреблении воды для пользователя и месяца</h3>

Meter userMeterData = service.getMeterForUserAndMonth(user, "month");
<br>
<h3>Получение списка всех пользователей</h3>

List<User> allUsers = service.getAllUsers();
<br>
<h3>Получение карты данных о потреблении воды для пользователя</h3>

Map<String, Meter> userMeterMap = service.getUserMeterMap(user);
<br>
<h3>Проверка существования данных для месяца у пользователя</h3>

boolean isMonthExist = service.isMonthExist(user, "month");
<br>
Пример инициализации данных
В конструкторе класса Service происходит инициализация данных для двух пользователей с образцовыми данными о потреблении воды в январе и феврале.

<h2>Примечание</h2>

Методы возвращают соответствующие значения или null/пустую коллекцию в случае отсутствия данных.
